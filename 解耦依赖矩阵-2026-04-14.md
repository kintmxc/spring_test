## 本轮收口点

1. ProductService 删除过渡性内部方法：
- getById
- decreaseStock
- increaseStock
- getImagesByProductId
- getProductsByIdsAndSaleStatus
- getTopProductsBySalesCount

2. 订单库存能力统一下沉到 AggregateQueryService：
- decreaseProductStockAndIncreaseSales
- increaseProductStockAndDecreaseSales

3. 关键路径结果：
- 订单创建扣库存与销量：OrderServiceImpl -> AggregateQueryService
- 订单取消回滚库存与销量：OrderServiceImpl -> AggregateQueryService
- 销量排行查询农户信息：SalesRankServiceImpl -> AggregateQueryService

4. 聊天模块额外收口：
- ChatServiceImpl 去除跨 Service 注入：FarmerService、UserService

5. 并发一致性加固：
- AggregateQueryServiceImpl 的回滚方法 `increaseProductStockAndDecreaseSales` 已从“先查后改”调整为单条 SQL 条件更新：
	- `stock = IFNULL(stock, 0) + quantity`
	- `sales_count = GREATEST(IFNULL(sales_count, 0) - quantity, 0)`
- 目的：缩小并发窗口，降低读写竞态导致的数据覆盖风险。
