# 小程序接口文档（最新版）

## 1. 基础信息

- Base URL: `http://11xq9976173yt.vicp.fun`
- 接口前缀: `/api`
- 完整示例: `http://11xq9976173yt.vicp.fun/api/products`
- 数据格式: `application/json`

## 2. 统一响应格式

后端当前统一返回如下字段：

```json
{
  "code": 0,
  "success": true,
  "message": "操作成功",
  "data": {}
}
```

- `code`: 业务码
  - `0` 成功
  - `400` 请求参数/业务错误
  - `401` 未登录或会话失效
  - `403` 无权限
  - `404` 资源不存在
  - `500` 服务器异常
- `success`: 兼容字段，旧前端仍可继续判断
- `message`: 提示文案
- `data`: 业务数据

## 3. 鉴权说明

项目当前采用 **Session 鉴权**（非 JWT）。

- 登录成功后，服务端通过 `Set-Cookie` 返回 `JSESSIONID`
- 小程序后续请求需携带 Cookie（建议统一处理）
- 需要登录的接口：除 `auth/sms`、`auth/register`、`auth/login`、`auth/phone-login`、`auth/wechat-login`、`auth/logout`、`ai/advice` 之外的 `/api/**`
- 角色说明：
  - `admin`：仅后台创建与登录，不走小程序注册
  - `consumer`：小程序普通用户
  - `farmer`：小程序农户用户

微信小程序请求建议：

```js
wx.request({
  url: 'http://11xq9976173yt.vicp.fun/api/auth/login',
  method: 'POST',
  data: { phone: '13800000000', code: '123456' },
  header: { 'content-type': 'application/json' },
  success(res) {
    // 记录返回 cookie，用于后续接口
  }
})
```

## 4. 认证模块

### 4.1 发送验证码
- `POST /api/auth/sms`
- 请求体：

```json
{ "phone": "13800000000" }
```

### 4.2 注册
- `POST /api/auth/register`
- 请求体：

```json
{
  "phone": "13800000000",
  "code": "123456",
  "nickName": "张三",
  "role": "consumer"
}
```

- `role` 仅支持 `consumer` 或 `farmer`
- 传 `role=admin` 会被拒绝（管理员账号仅后台创建）

### 4.3 手机号验证码登录（小程序推荐）
- `POST /api/auth/phone-login`

```json
{
  "phone": "13800000000",
  "code": "123456",
  "role": "consumer"
}
```

或

```json
{
  "phone": "13900000001",
  "code": "123456",
  "role": "farmer"
}
```

- 测试阶段固定验证码：`123456`

### 4.4 微信快速登录（预留）
- `POST /api/auth/wechat-login`

```json
{
  "code": "wx_test_code_001",
  "nickName": "微信用户A",
  "avatar": ""
}
```

### 4.5 账号密码登录（后台/兼容）
- `POST /api/auth/login`
- 入参示例：

```json
{ "username": "admin", "password": "123456" }
```

### 4.6 当前用户
- `GET /api/auth/me`

返回字段（当前版本统一）：
- `id` / `userId`
- `username`
- `roleCode` / `role`
- `name` / `nickName` / `realName`
- `phone`
- `avatar`
- `address`

### 4.7 退出登录
- `POST /api/auth/logout`

### 4.8 登录成功返回重点字段

```json
{
  "code": 0,
  "success": true,
  "message": "登录成功",
  "data": {
    "token": "sessionId",
    "roleCode": "CONSUMER",
    "userInfo": {
      "id": 1,
      "username": "consumer01",
      "realName": "测试用户",
      "nickName": "测试用户",
      "roleCode": "CONSUMER",
      "role": "consumer",
      "phone": "13800000000",
      "avatar": "",
      "address": ""
    }
  }
}
```

- 前端建议优先使用 `data.userInfo.role` 做页面分流
- 兼容字段 `data.roleCode` 可作为兜底
- `data.userInfo` 与 `GET /api/auth/me` 的名称语义已对齐（`name/nickName/realName`）

### 4.9 管理员创建与登录说明

- 管理员账号不通过小程序注册
- 管理员由后台或数据库创建
- 管理员可用 `/api/auth/login` 进行账号密码登录

### 4.10 历史兼容说明

- 之前文档中 `phone/code` 直接调 `/api/auth/login` 的方式已不推荐
- 小程序请改用 `/api/auth/phone-login`

### 4.11 注册示例（农户）

```json
{
  "phone": "13900000003",
  "code": "123456",
  "nickName": "果园老李",
  "role": "farmer"
}
```

## 5. 商品模块

### 5.1 商品列表
- `GET /api/products`
- 查询参数：
  - `page`（可选，映射页码）
  - `pageSize`（可选）
  - `keyword`（可选，映射商品名搜索）
  - `categoryId`（可选）
  - `farmerId`（可选）
  - `saleStatus`（可选）
  - `sort`（可传，当前后端预留）

### 5.2 商品详情
- `GET /api/products/{id}`

### 5.3 发布商品
- `POST /api/products`
- 请求体（示例）：

```json
{
  "farmerId": 1,
  "categoryId": 2,
  "productName": "红富士苹果",
  "price": 25.0,
  "stock": 50,
  "unitName": "斤",
  "originPlace": "陕西洛川",
  "coverImage": "/uploads/images/xxx.jpg",
  "description": "新鲜采摘",
  "saleStatus": 1
}
```

### 5.4 更新商品
- `PUT /api/products/{id}`

### 5.5 上下架
- `PUT /api/products/{id}/sale-status`
- 请求体：

```json
{ "status": 1 }
```

### 5.6 修改库存
- `PUT /api/products/{id}/stock`
- 请求体：

```json
{ "stock": 50 }
```

### 5.7 删除商品
- `DELETE /api/products/{id}`

## 8. 订单模块

### 8.1 创建订单
- `POST /api/orders`
- 请求体：

```json
{
  "productId": 1,
  "quantity": 2,
  "addressId": 1,
  "remark": "请尽快发货"
}
```

- 成功 `data` 示例：

```json
{ "orderId": 1001 }
```

### 8.2 订单列表
- `GET /api/orders`
- 查询参数：
  - `page`
  - `pageSize`
  - `status`（映射订单状态）

### 8.3 订单详情
- `GET /api/orders/{id}`

### 8.4 更新状态
- `PUT /api/orders/{id}/status`
- 请求体示例：

```json
{ "targetStatus": 1, "remark": "支付完成" }
```

### 8.5 发货
- `PUT /api/orders/{id}/ship`
- 请求体兼容：

```json
{
  "logisticsCompany": "顺丰速运",
  "trackingNo": "SF123456789",
  "shipRemark": "已发货"
}
```

或

```json
{
  "companyName": "顺丰速运",
  "trackingNo": "SF123456789",
  "shipRemark": "已发货"
}
```

### 8.6 取消订单
- `POST /api/orders/{id}/cancel?remark=不想要了`
